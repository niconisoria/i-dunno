## Brainstorm

Send email on key events: signup, payment, password reset.

Related: none

## Story

As user, want email on key events, so stay informed.

AC:
1. Email sent on signup
2. Email sent on payment success
3. Email sent on password reset request
4. Emails use app branding

## Design

### Flow

```
┌───────┐     ┌────────────┐     ┌──────────────┐
│ event │ ──▶ │ build mail │ ──▶ │ send via SMTP│
└───────┘     └────────────┘     └──────────────┘
                    │
              ┌──────────┐
              │  log err │
              └──────────┘
```

### Data

```
input:  { event: string, user: { email: string, name: string } }
output: { sent: bool, message_id: string }
```

### Modules

- `app/mailer.rb` — create, one method per event type
- `test/mailer_test.rb` — create
